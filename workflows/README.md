# Workflows

Markdown SOPs that define how to accomplish objectives using the WAT framework.

## Structure of a Workflow

Each workflow should include:

- **Objective** — what this workflow accomplishes
- **Inputs** — what's required before starting
- **Tools** — which scripts in `tools/` to call and in what order
- **Outputs** — what gets produced and where it goes
- **Edge Cases** — known failure modes and how to handle them

## Adding a New Workflow

1. Create `workflows/your_workflow_name.md`
2. Follow the structure above
3. Update this README with a one-line description
